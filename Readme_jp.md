# Carrier Codex（キャリア・コーデックス）

**10年生（高校1年相当）修了後、理系・商系・文系の進路を選ぶインドの学生のための、データ駆動型キャリア分析プラットフォーム。**

Carrier Codex は、当て推量による進路選択を実際の調査データに置き換えるプロジェクトです。給与帯、市場の飽和度、後悔率、進路選択の心理、キャリアの軌跡——これらはすべて、一般的な進路カウンセラーの決まり文句ではなく、自ら作成した Jupyter ノートブックから直接抽出されています。さらに、キャリア関連の会話のみに厳格に制限された AI キャリアアドバイザーも搭載しています。

## このプロジェクトについて

インドでは、10年生を修了した直後に、学生は理系（Science）・商系（Commerce）・文系（Arts）のいずれかの進路を選ばなければなりません。多くの場合、その選択は実データではなく、家族の意見や友人からの圧力、あるいは「トップ層」の逸話的な成功談に基づいて行われています。Carrier Codex は、この決断の前に実際の数字——給与の幅、職業ごとの市場飽和度、後悔に関する統計、そして学生がなぜその選択をするのかという心理的背景——を提示するために作られました。

本プロジェクトは **4つの調査ノートブック**（`science_stream_second_page.ipynb`、`commerce_main_page.ipynb`、`arts.ipynb`、`decision.ipynb`）を中心に構築されています。各ノートブックには、マークダウンによる分析、データセット、matplotlib による可視化が含まれています。このWebアプリは、その調査結果を忠実に、かつ厳密にソースに基づいてインタラクティブなサイトへと翻訳したものであり、ノートブックに含まれる内容を超えて創作されたデータは一切ありません。

## ページごとの詳細

### `/` — ホーム
プロジェクト名を表示するフルビューポートのヒーローセクションから始まり、スクロールに連動したテキストモーフ演出があります。**「Decide Your Career」** がフェードアウト・ぼかし処理されながら **「Career Analytics」** がフェードインしていく仕組みで、`home.js` が `window.scrollY` を500pxのモーフ範囲に対して読み取ることで制御されています。その下には、`IntersectionObserver` を利用してスクロールに応じて表示される3枚のカードがあり、このプロジェクトが存在する理由、何を行うのか、誰が作ったのかを説明しています。ページの最後には、理系・商系・文系の各ページへのリンクとなる3枚のカードが配置されています。

### `/science` — 理系（Science）
`science_stream_second_page.ipynb` から厳密に抽出したデータに基づいています。以下を扱います：
- インドにおける科目の組み合わせの自由度を、イギリス・アメリカ・カナダ・オーストラリア・ドイツ・シンガポールと比較（ドーナツチャート）
- PCM（物理・化学・数学）、PCB（物理・化学・生物）、PCMB の給与分布——最低・中間・最高（グループ化棒グラフ）
- PCB/PCM を切り替えられるキャリア探索機能で、職業ごとの現在の給与と将来予測の給与を表示（横棒グラフ）
- 市場飽和度に関する考察（志願者数 対 実働中の専門職人数）
- 「普通の学生」と「自覚的な学生」の努力対成果を比較した散布図

### `/commerce` — 商系（Commerce）
`commerce_main_page.ipynb` から厳密に抽出したデータに基づいています。以下を扱います：
- 進路の分布（文系45% / 理系35% / 商系20%）と、なぜ商系が最も少ないのか
- 学生が実際に商系を選ぶ理由（消極的選択、難関資格突破狙い、数学からの逃避、家業のため、純粋な興味）
- 5つの「誤解と現実」の対比（「妥協の選択」という誤解、「楽な進路」という思い込み、AIによる代替への不安など）
- 伝統的な商系キャリアと現代的な商系キャリアの比較——雇用の安定性、給与の伸び、グローバルな流動性、リモートワークの観点でスコア化

### `/arts` — 文系（Arts）
`arts.ipynb` から厳密に抽出したデータに基づいています。以下を扱います：
- 学生が文系を選ぶ理由（UPSC（連邦公務員試験）志望、消極的選択、数学・理科からの逃避、法律、純粋な情熱）
- 3つの「誤解と現実」の対比（「落ちこぼれの行き先」という誤解、「就職できない」という思い込み、AI時代における文系の現実）
- 文系×テクノロジーの融合による現代的なキャリア（UXリサーチ、行動データサイエンス、対話型AI設計など）と、それぞれの成長スコア
- 伝統的な文系キャリアパスとその飽和率（UPSCの脱落率は99.9%）

### `/about` — 制作者について
あなた（Monu Verma）についての紹介、GitHub・LinkedIn へのリンク、使用した技術スタック、そして他の全ページへのクイックリンクが掲載されています。

### `/chat` — AI キャリアチャット
Groq の `llama-3.3-70b-versatile` を活用したチャットインターフェースで、`config.py` 内のシステムプロンプトによってキャリア相談のみに厳密に制限されています。会話履歴はセッションごとに SQLite（`database/chat_history.db`）に保存されます。

## データパイプライン — ノートブックからブラウザへ

1. **一次情報源**：4つの `.ipynb` ノートブックには、マークダウンによる解説、pandas のデータフレーム、matplotlib によるチャート出力が含まれています。
2. **抽出**：マークダウンセルとデータフレームの出力は、各ノートブックの JSON 構造から直接読み取られ、整形された JSON ファイルへと書き起こされました——ノートブックに含まれていない数値が創作・推測されることは一切ありません。
3. **保存**：`static/data/{science,commerce,arts}_data.json` に、フロントエンドが扱いやすいフラットな形式で保存されています。
4. **描画**：各進路専用の JavaScript ファイル（`science.js`、`commerce.js`、`arts.js`）が `DOMContentLoaded` 時に対応する JSON を取得し、共通の `CodexCharts` 描画関数に渡すとともに、誤解カードやインサイト欄、キャリアカードの HTML を挿入します。

この仕組みにより、チャートや統計値の更新は該当する JSON ファイルを編集するだけで完結し、HTML や JavaScript の変更は不要です。

## チャート描画エンジン

`static/js/charts.js` は `CodexCharts` というグローバルオブジェクトを公開しており、`donut`（ドーナツ）、`groupedBar`（グループ化棒グラフ）、`horizontalBar`（横棒グラフ）、`scatter`（散布図）の4種類のチャートに対応しています。

**なぜ Chart.js を使わないのか？** 本プロジェクトの初期バージョンでは、CDN（`cdnjs.cloudflare.com`）経由で Chart.js を読み込んでいました。しかし実際には、CDN へのリクエストが失敗した、ブロックされた、あるいはページ自身のスクリプトより読み込みが遅れた・順序が前後したなどの理由により、チャートが正しく表示されませんでした。プロジェクトを完全に自己完結させ、ネットワーク障害の影響を受けないようにするため、チャート描画層全体を**外部依存ゼロの素の Canvas 2D 描画コード**として書き直しました。ドーナツ、棒グラフ、横棒グラフ、トレンドライン付きの散布図に至るまで、すべてのチャートは `CanvasRenderingContext2D` の基本機能（`arc`、`fillRect`、`moveTo`/`lineTo` など）を使って手作業で描画されており、編集的でダーク／ペーパー調のテーマ（インク色 `#1a1a1a`、紙色 `#fdfcfa`、ミスト色 `#d4cfc7`、各進路ごとのアクセントカラー）に合わせてスタイリングされています。

これにより、**ページ自体の読み込みが完了した後は、チャートの表示にインターネット接続を必要としません**——プレゼンテーションなどでオフラインでデモを行う際の信頼性が大きく向上します。

## セットアップとインストール

### 前提条件
- Python 3.10 以上
- pip

### 手順

```bash
# 1. プロジェクトをクローン、または展開する
cd carrier-codex

# 2.（推奨）仮想環境を作成する
python -m venv venv
source venv/bin/activate      # Windowsの場合: venv\Scripts\activate

# 3. 依存パッケージをインストールする
pip install -r requirements.txt
```

`requirements.txt` の内容：
```
Flask==3.0.3
groq==0.11.0
python-dotenv==1.0.1
```

## 環境変数

AI チャット機能を使用するには、**無料の Groq API キー**が必要です。

1. [console.groq.com](https://console.groq.com) にアクセスし、サインアップしてください（無料プランで十分です）。
2. API キーを作成します。
3. プロジェクトルートにある `.env` を開き、キーを貼り付けます：

```env
# このファイルは絶対にコミットしないでください。すでに .gitignore に登録されています。
GROQ_API_KEY=your_groq_api_key_here
```

このキーが未設定のままでも、チャット機能がアプリ全体をクラッシュさせることはありません——キーがまだ設定されていない旨をユーザーに伝える親切なメッセージを返すだけなので、サイトの他の部分は問題なく利用できます。

## AI チャット機能

- **モデル**：Groq の OpenAI 互換チャット補完 API 経由で `llama-3.3-70b-versatile` を使用。
- **システムプロンプト**（`config.py` 内の `AI_SYSTEM_PROMPT`）：進路選択、職種、必要なスキル、給与、教育の道筋、インドにおけるキャリアプランニングといった、キャリアに関連する話題のみに厳密に限定されています。話題から外れた質問は丁寧に断り、キャリアの話題へと誘導します。
- **永続化**：すべてのユーザー／アシスタントのメッセージのやり取りは `chat_messages` テーブル（`session_id`、`role`、`content`、`created_at`）に保存され、ブラウザ側でランダムに生成されるセッションID（`chat.js`）に紐づけられます。
- **API エンドポイント**：
  - `POST /api/chat/send` — メッセージを送信し、応答を取得、両方のやり取りを保存
  - `GET /api/chat/history/<session_id>` — 該当セッションの全履歴を取得

## 既知の制限事項・今後の課題

- **Decision（決断）ノートブック**（`decision.ipynb`）——進路選択の心理学、後悔率、ハイブリッドキャリアのデータセットを扱う内容——は、まだ専用ページとして構築されていません。現在の計画では、この内容は専用の「Decision」セクションとして組み込むか、AI チャットの文脈情報として直接活用することを想定しています。
- `about.html` には現状、プレースホルダーの GitHub／LinkedIn の URL（`yourusername`）が入っているため、公開前に実際のプロフィールリンクへ差し替える必要があります。
- 自動テストスイートはまだ存在しません——現時点での動作確認は、ルートごとの手動テスト（Flask のテストクライアントを使用し、全6ページが HTTP 200 を返すことを確認）にとどまっています。
- 現状、チャットはどの進路のデータセットも文脈として参照していないため、その回答はあなたのノートブックの具体的な数値に基づいたものではなく、システムプロンプトによって方向づけられた一般的な LLM の知識によるものです。`science_data.json` / `commerce_data.json` / `arts_data.json` をチャットの文脈情報として組み込めば、あなたの実際のデータに基づいた回答ができるようになります。
